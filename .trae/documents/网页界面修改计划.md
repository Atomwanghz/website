# 网页界面修改计划

## 1. 修改页脚背景颜色

### 目标
将页面底部的页脚元素背景颜色从当前的蓝色调整为纯黑色

### 修改位置
- 文件：`/home/atomwanghz/my-blog/node_modules/hexo-theme-butterfly/source/css/_layout/footer.styl`
- 行号：第3行
- 当前代码：`background-color: $light-blue`
- 修改后：`background-color: #000000`

### 修改说明
- 直接将背景色从主题变量 `$light-blue` 修改为纯黑色 `#000000`
- 确保页脚文字颜色与黑色背景形成良好对比度

## 2. 移除清单组件中的列表项

### 目标
在页面清单组件中，移除"相册"和"游戏demo"两个列表项

### 修改位置
- 文件：`/home/atomwanghz/my-blog/_config.butterfly.yml`
- 行号：第20-24行的menu配置

### 修改前
```yaml
menu:
  首页: / || fas fa-home
  清单 || fas fa-list:
    相册: /gallery/ || fas fa-images
    游戏Demo: /games/demo/ || fas fa-gamepad
```

### 修改后
```yaml
menu:
  首页: / || fas fa-home
  清单 || fas fa-list:
```

### 修改说明
- 直接删除"相册"和"游戏Demo"两个子项
- 保留"清单"父项，确保菜单结构完整

## 3. 验证修改效果

### 步骤
1. 运行 `npm run clean` 清理缓存
2. 运行 `npm run build` 重新构建项目
3. 运行 `npm run server` 启动本地服务
4. 访问 `http://localhost:4000` 查看修改效果

### 验证要点
- 页脚背景颜色变为纯黑色
- 清单菜单中不再显示"相册"和"游戏Demo"子项
- 页面布局保持完整，无视觉错位
- 所有交互元素正常响应

## 4. 注意事项

- 确保修改的CSS文件路径正确
- 修改配置文件时注意YAML语法格式
- 验证修改后所有页面都能正常访问
- 确保响应式布局在不同设备上都能正常显示
